Hard restrictions:
- Do not emit imports.
- Do not emit Python functions.
- Do not emit classes.
- Do not emit loops.
- Do not emit if/elif/else statements.
- Do not emit executable runtime code.
- Do not call arbitrary helper functions.
- Do not model concrete file paths or raw file I/O.
- For standalone variables declared with var("name", Type), references in predicates must use Var("name").
- For standalone variable updates, use Set("name", value). Do not write Set(Var("name"), value).

Modeling rules:
- Declare workflow state using entity(...) records or var(...).
- Initialize every declared field and variable in init(...).
- Convert workflow steps into action(...) declarations.
- Put current-state guards in requires=[...].
- Put state updates in changes=[...].
- Model uncertain external outcomes as Bool or Enum state fields and separate guarded actions.
- Include properties for completion, retries, fallback, invariants, or success reachability when the markdown states them.
- If the markdown describes a missing branch or unreachable success, preserve that bug in the DSL instead of inventing the missing action.
- If behavior is ambiguous, model nondeterminism explicitly rather than assuming constants.
- Use init(...) only for valid starting states, not to enumerate all workflow states.
- Do not encode uncertain future outcomes as fixed booleans in init(...).
- Use AnyOf(Bool) (or AnyOf(Enum(...))) in init(...) when a field represents an uncertain external outcome; this lets TLC explore both branches instead of freezing one.
- If a step may succeed or fail, model separate actions or explicit branching in actions.
- Do not guard creation/start actions on fields of an entity whose existence is false.
- Prefer canonical start states over placeholder record values that contradict existence flags.
- Correct examples:
  - Predicate reference: Var("order_exists")
  - Predicate equality: Eq(Var("order_exists"), False)
  - State update: Set("order_exists", True)
  - Forbidden property: forbidden("cannot_cancel_paid_order", when=And(...))
- Incorrect examples:
  - Eq("order_exists", False)
  - Not("order_exists")
  - Set(Var("order_exists"), True)
  - forbidden("cannot_cancel_paid_order", And(...))

Exploration-safe examples:
```python
# Bad
var("payment_succeeds", Bool)
init(
    Eq(Var("payment_succeeds"), False),
)
action("payment_attempt_succeeds", requires=[Var("payment_succeeds")], changes=[...])
action("payment_attempt_fails", requires=[Not(Var("payment_succeeds"))], changes=[...])
```

```python
# Better
action("payment_attempt_succeeds", requires=[...], changes=[...])
action("payment_attempt_fails", requires=[...], changes=[...])
```

```python
# Bad
action(
    "create_order",
    requires=[
        Not(Var("order_exists")),
        Gt(Field("order", "amount"), 0),
    ],
    changes=[...],
)
```

```python
# Better
action(
    "create_order",
    requires=[Not(Var("order_exists"))],
    changes=[
        Set("order_exists", True),
        SetField("order", "amount", 1),
    ],
)
```
